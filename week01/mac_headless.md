## 一、完成了什么：将一台桌面 Mac 改造成无人值守的 AI 网关服务器

这次实践的核心方法借鉴自 Niu Baokang 同学的无头部署方案，我在其基础上进行了适配和扩展。

表面上是在配置远程控制，本质上完成了一次“桌面设备去桌面化”的系统工程。把一台 Mac mini 从“需要显示器、键盘、鼠标的交互式个人电脑”，改造成了一台可以拔掉所有外设、藏在角落里、7×24 小时静默运行的 AI 服务节点。这台机器上运行着 Hermes Gateway，作为个人与大模型之间的固定桥梁——随时随地用 Windows 电脑接入，就能直通硅基流动的云端算力。

牛保康同学的原始方案采用 Hermes 自带的飞书通道作为交互入口，SSH 作为备援排查手段。我在实践中进行了关键扩展：引入 RustDesk 构建了双通道远程操控架构——

- 主通道（应用层）：沿用牛保康的方案，通过 Hermes 飞书通道进行日常 AI 交互，轻量、无需额外工具。
- 备援通道（系统层）：新增 RustDesk 跨平台远程桌面，让 Windows 电脑能直接操控 Mac 的完整图形界面。这个补充弥补了原方案中 SSH 命令行排错门槛较高的问题，确保当应用层通道出问题时，可以直观地通过桌面环境进行排查和恢复。

这不是简单的“装了个远程软件”，而是在牛保康方案的基础上，构建了一套从应用入口到底层系统的分级冗余访问体系。

## 二、遇到的理解问题：无头部署中“系统级睡眠锁”的认知盲区

牛保康同学整理的核心系统设置，每一条都是在解除一个会阻止无头运行的“睡眠锁”，我在实践过程中逐一验证并加深了理解：

### 1. 自动登录的意义

无头 Mac 重启后（断电、更新等），如果停在登录界面，所有后台服务都不会启动，Hermes 也不会运行。自动登录确保了机器只要通电开机，就能一路跑到桌面并拉起所有服务——这是无人值守的第一前提。

### 2. FileVault 是最隐蔽的陷阱

磁盘加密在安全上是好东西，但在无头场景下是致命锁。一旦 FileVault 开启，Mac 通电重启后会停在解锁磁盘的画面上，不输入密码连系统都进不去，自动登录形同虚设。牛保康在方案中特别标注了“最重要”，我在实践后才真正理解——关闭它本质上是拆除了一条会阻止系统自举全程无人化的硬检查点。安全加密和无人值守是一对需要刻意取舍的矛盾。

### 3. 电源管理命令的深层含义

`sudo pmset -a sleep 0 displaysleep 0 disksleep 0` 看似只是不让电脑休眠，实则是强制将这台 Mac 从“节能优先的移动设备行为模式”，切换为“持续运行的服务节点行为模式”。`tcpkeepalive 1` 确保网络连接不因闲置而断开，`autorestart 1` 保证意外断电后能自动重启——这些组合起来等于给这台机器装了一个软件层面的“看门狗”机制。

### 4. 我对原方案的改进：用 RustDesk 替代纯 SSH 备援

牛保康同学的方案以 SSH 远程登录作为备用排查手段，这在命令行熟练的情况下非常高效。但我在实践中意识到，当 Hermes 本身或网络通道出问题时，纯命令行排错有一定门槛。因此我引入 RustDesk，提供了一条与 Hermes 完全解耦的、图形化的独立控制路径。即便 Hermes 崩溃、飞书断连，我依然能从 Windows 看到一个完整的 Mac 桌面并像本地一样操作。这是对原方案在运维易用性上的补充——让“系统级逃生通道”从命令行下沉到图形界面，降低了紧急排错时的认知负担。

## 三、学到的关键点：可复用的“去桌面化部署”元方法论

### 牛保康的“开机即服务”三把锁清单

要让一台 Mac 变成无头服务器，必须依次解除三道锁：

- 登录锁（自动登录）
- 加密锁（关闭 FileVault）
- 睡眠锁（`pmset` 全能电源配置）

这是一份可迁移清单，未来任何时候面对 Mac 的无头部署，都照此执行。

### 冗余通道的层级分离原则

我在牛保康方案基础上构建的架构中，主通道（飞书/Hermes 交互）和备援通道（RustDesk 远程桌面）运行在两个完全独立的技术栈上，互不依赖。这种“完全解耦的双通道”设计确保了一个通道的故障不会阻断另一个通道的可用性，是服务高可用性的核心思想。这里同样可以迁移到 OpenClaw 等其他类似工具。

### 用 RustDesk 构建“零配置 P2P 跨平台远程桌面”

这是我对原方案的关键补充。相比 VNC 需要处理 IP、端口转发和网络拓扑的麻烦，RustDesk 的 P2P 直连特性让 Windows 和 Mac 之间的远程桌面搭建，从“一次网络配置工程”简化为“一次应用安装”，几乎不需要理解底层网络细节。

### “运维入口下沉”的智慧

牛保康的 SSH 方案是专业的命令行排障路径，而我在其基础上增加 RustDesk 图形界面通道，本质上是将运维门槛从“必须熟悉命令行”下沉到“所见即所得操作”。这不是替代，而是分层——日常优先用飞书，紧急情况用 RustDesk 图形化排查，深度排错仍可走 SSH。这是工程实用性对技术纯粹性的理性补充，也是根据不同场景选择最合适入口的思维体现。
